ETO Newsletter Sender 사용설명서

주홍콩경제무역대표부(도쿄) — 월간 한국어 뉴스레터 발송 프로그램 · 2026-07 기준

1. 이 프로그램이 하는 일

구독자 명단(CSV)을 불러와 MailerLite를 통해 월간 뉴스레터 이메일을 보내는 Windows 프로그램입니다. 발송 전에 반드시 ① 명단 대조 ② 테스트 메일 ③ 확인 체크를 거치도록 되어 있고, 마지막에도 5초의 취소 기회가 있어 실수로 전체 발송되는 사고를 막아 줍니다.

2. 처음 한 번만 하는 설치

  1. Python 설치 — python.org/downloads에서 최신 버전 설치. 설치 첫 화면에서 "Add Python to PATH" 체크박스를 꼭 켜세요.
  2. Git 설치 — git-scm.com/download/win에서 설치 (모든 옵션 기본값 그대로 Next).
  3. 프로그램 받기 — 원하는 폴더에서 주소창에 cmd를 입력해 명령창을 열고:
    git clone https://github.com/Ko0orean/eto-newsletter-sender.git
  4. 설정 파일 받기 — 담당자에게 config.json(API 키 포함)과 구독자 CSV 파일을 USB 등 안전한 방법으로 받아, config.json을 프로그램 폴더(README.md가 있는 곳)에 넣으세요.
보안 주의 — config.json과 구독자 CSV는 이메일·메신저로 보내지 마세요. API 키가 유출되면 구독자 전체 명단이 노출되고 누구나 발송할 수 있게 됩니다.

3. 매달 발송 절차

  1. 시작 — 폴더 안의 Update ETO Newsletter.bat를 더블클릭합니다. 최신 버전을 자동으로 받고 프로그램이 열립니다.
  2. 명단 업로드 — Upload list… 버튼으로 이번 달 구독자 CSV를 엽니다. (한국어 Excel로 저장한 CSV도 그대로 열립니다.)
  3. 명단 대조 — Compare with MailerLite 버튼을 누르면 지난달 발송 명단과 비교해 신규 구독 / 탈퇴 / 수신거부를 보여줍니다. (활용법은 4장 참고)
  4. 뉴스레터 선택 — Browse…로 Word(.docx) 파일을 선택하고, 웹사이트에 올린 PDF 주소를 PDF link 칸에 붙여넣습니다. Preview 버튼을 누르면 실제 발송될 모습이 브라우저로 열립니다.
  5. 제목 확인 — Subject 칸의 제목을 이번 달에 맞게 고칩니다 (지난번 제목이 기억되어 있습니다).
  6. 테스트 발송 — Test email(s) 칸에 본인 주소(여러 명이면 쉼표로 구분)를 넣고 Send test. 받은 메일이 Outlook·휴대폰에서 제대로 보이는지 확인하세요.
  7. 확인 체크 — 대조와 테스트가 모두 끝나면(✓ 표시) "I reviewed the comparison and the test email" 체크박스가 켜집니다. 체크해야 발송 버튼이 풀립니다.
  8. 발송 — Send to all subscribers 1차 클릭 → 표에서 제외할 사람이 있으면 체크 해제 → Confirm — send now 2차 클릭 → 확인창에서 Yes → 5초 카운트다운 후 발송됩니다. 카운트다운 중 빨간 버튼을 누르면 취소됩니다.
발송이 끝나면 MailerLite의 구독자 그룹이 이번 발송 명단으로 자동 정리되어, 다음 달 대조의 기준이 됩니다.

4. 명단 대조 화면 활용

항목의미할 수 있는 것
New sign-upsCSV에는 있지만 MailerLite 그룹에 없는 신규 구독자체크 후 Add checked to “ETO Korea Newsletter Subscribers” — 그룹에 미리 등록
Departed그룹에는 있지만 이번 CSV에서 빠진 사람 체크 후 Add checked to the current list(명단에 복원, 새 CSV로 저장) 또는 Delete checked from MailerLite… (계정에서 삭제)
Unsubscribed메일의 수신거부 링크를 직접 누른 사람 정보 표시용. CSV에 남아 있어도 MailerLite가 자동으로 발송에서 제외합니다. 임의로 되살리지 마세요.

5. 뉴스레터 Word 파일 작성 요령

  • 제목은 글자 크기를 키우는 대신 Word의 스타일(제목 1, 제목 2)을 사용하세요. 그래야 이메일에서도 제목으로 표시됩니다.
  • 굵게, 기울임, 글머리 기호 목록, 표는 그대로 이메일에 반영됩니다.
  • 문서에 붙여넣은 이미지는 Gmail·Outlook에서 표시되지 않는 경우가 많습니다. 이미지가 필요하면 웹사이트에 올린 뒤 링크로 넣고, 반드시 테스트 메일에서 확인하세요. (이미지가 감지되면 프로그램이 경고를 띄웁니다.)
  • 전체 디자인(상단 남색 제목 띠, PDF 다운로드 버튼, 하단 SNS·수신거부 배너)은 프로그램이 자동으로 입힙니다.

6. 자주 나오는 메시지 / 문제 해결

상황대처
발송 버튼이 눌리지 않음정상입니다 — 명단 대조와 테스트 발송을 먼저 마치고 확인 체크박스를 켜야 풀립니다. 뉴스레터 파일이나 제목을 고치면 테스트 ✓가 다시 꺼지므로 테스트를 새로 보내야 합니다.
"The PDF link does not work"PDF 주소가 잘못됐거나 아직 업로드 전입니다. 주소를 고치면 됩니다. 아무것도 발송되지 않았으니 안심하세요.
"Possible duplicate send" 경고같은 제목으로 최근에 이미 발송된 기록이 있다는 뜻입니다. 이번 달 두 번째 발송이 아닌지 확인하고, 확실할 때만 Yes를 누르세요.
"SAFETY STOP" 에러수신자 설정이 확인되지 않아 발송을 중단한 것입니다(전체 오발송 방지 장치). 담당자에게 알려주세요.
업데이트(bat) 실패인터넷 연결을 확인하고 다시 실행하세요. "local changes" 문구가 보이면 담당자에게 연락하세요.
인증 실패 (401)Settings의 API 키가 비었거나 틀렸습니다. Settings → Test connection으로 확인하세요.
사내망에서 연결 오류(SSL)임시로 Settings → Skip SSL verification을 켤 수 있으나, IT에 프록시 인증서를 요청하는 것이 올바른 해결입니다.

7. 하지 말아야 할 것

  • 프로그램 폴더 안의 파일을 직접 수정·삭제하지 마세요 (CSV, config.json 제외). 코드 수정은 담당자가 GitHub로 배포합니다.
  • config.json·구독자 CSV를 외부로 보내거나 공유 폴더에 두지 마세요.
  • MailerLite 웹사이트에서 그룹을 직접 지우지 마세요 — 다음 달 명단 대조 기준이 사라집니다.
